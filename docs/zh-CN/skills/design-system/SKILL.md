# 设计系统 — 生成与审查视觉系统

## 何时使用

- 启动需要设计系统的新项目
- 对现有代码库进行视觉一致性审查
- 重新设计前——了解当前状态
- 当 UI "看起来有问题"但无法精确定位原因时
- 审阅涉及样式修改的 PR

## 工作原理

### 模式一：生成设计系统

分析代码库并生成一套统一的设计系统：

```
1. 扫描 CSS/Tailwind/styled-components 中的现有模式
2. 提取：颜色、字体、间距、圆角、阴影、断点
3. 通过浏览器 MCP 研究 3 个竞品网站以获取灵感
4. 提出设计令牌集（JSON + CSS 自定义属性）
5. 生成 DESIGN.md，说明每项决策的理由
6. 创建交互式 HTML 预览页面（独立文件，无依赖）
```

输出：`DESIGN.md` + `design-tokens.json` + `design-preview.html`

### 模式二：视觉审查

从 10 个维度对 UI 进行评分（每项 0-10 分）：

```
1. 颜色一致性 — 是否在使用调色板，还是随意使用十六进制色值？
2. 字体层级 — h1 > h2 > h3 > 正文 > 说明文字是否清晰？
3. 间距节奏 — 是否使用一致的间距比例（4px/8px/16px）？
4. 组件一致性 — 相似元素外观是否相似？
5. 响应式表现 — 在各断点下是否流畅或存在错位？
6. 深色模式 — 是否完整，还是只做了一半？
7. 动画效果 — 是否有目的性，还是滥用？
8. 无障碍性 — 对比度、焦点状态、触控目标区域
9. 信息密度 — 是否拥挤或简洁？
10. 精细度 — 悬停状态、过渡效果、加载状态、空状态
```

每个维度包含评分、具体示例以及精确到文件行号的修复建议。

### 模式三：AI 滥用检测

识别通用 AI 生成的设计模式：

```
- 到处滥用渐变效果
- 紫色到蓝色的默认配色
- 毫无目的的"玻璃拟态"卡片
- 不该有圆角的地方也加了圆角
- 滚动时的过度动画
- 带文字居中叠加渐变的通用 hero 区域
- 无个性的无衬线字体堆叠
```

## 示例

**为 SaaS 应用生成设计系统：**
```
/design-system generate --style minimal --palette earth-tones
```

**审查现有 UI：**
```
/design-system audit --url http://localhost:3000 --pages / /pricing /docs
```

**检测 AI 滥用：**
```
/design-system slop-check
```
