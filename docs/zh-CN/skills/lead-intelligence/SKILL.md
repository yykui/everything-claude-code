# 潜在客户情报

由 Agent 驱动的潜在客户情报流水线，通过社交图谱分析和温暖路径发现来查找、评分并触达高价值联系人。

## 何时激活

- 用户希望在特定行业寻找潜在客户或目标联系人
- 为合作、销售或融资建立外联名单
- 研究联系对象及最佳触达路径
- 用户说"寻找潜在客户"、"外联名单"、"我应该联系谁"、"温暖介绍"
- 需要按相关性对联系人列表进行评分或排名
- 希望梳理共同联系人以寻找温暖介绍路径

## 工具要求

### 必需
- **Exa MCP** — 对人物、公司和信号进行深度网络搜索（`web_search_exa`）
- **X API** — 关注者/关注图谱、共同分析、近期活动（`X_BEARER_TOKEN`，以及写入上下文凭据如 `X_CONSUMER_KEY`、`X_CONSUMER_SECRET`、`X_ACCESS_TOKEN`、`X_ACCESS_TOKEN_SECRET`）

### 可选（增强结果）
- **LinkedIn** — 如有直接 API 可用，否则通过浏览器控制进行搜索、简介查看和草稿撰写
- **Apollo/Clay API** — 如用户有访问权限，用于富化交叉参考
- **GitHub MCP** — 用于以开发者为中心的潜在客户筛选
- **Apple Mail / Mail.app** — 起草冷邮件或暖邮件，不自动发送
- **浏览器控制** — 当 API 覆盖范围不足或受限时，用于 LinkedIn 和 X

## 外联前的声音确立

不要用通用销售文案起草对外信息。

只要用户的声音很重要，就先运行 `brand-voice`。复用其生成的 `VOICE PROFILE`，而非在本技能中临时推导风格。

如果有实时 X 访问权限，起草前先抓取近期原创帖子。如果没有，使用提供的示例或可用的最佳仓库/网站素材。

## 第一阶段：信号评分

在目标垂直领域搜索高信号人员，并根据以下维度分配权重：

| 信号 | 权重 | 来源 |
|------|------|------|
| 职位/头衔匹配度 | 30% | Exa、LinkedIn |
| 行业匹配度 | 25% | Exa 公司搜索 |
| 主题近期活跃度 | 20% | X API 搜索、Exa |
| 粉丝数/影响力 | 10% | X API |
| 地理位置接近度 | 10% | Exa、LinkedIn |
| 与您内容的互动 | 5% | X API 互动 |

### 信号搜索方法

```python
# 第一步：定义目标参数
target_verticals = ["prediction markets", "AI tooling", "developer tools"]
target_roles = ["founder", "CEO", "CTO", "VP Engineering", "investor", "partner"]
target_locations = ["San Francisco", "New York", "London", "remote"]

# 第二步：使用 Exa 深度搜索人物
for vertical in target_verticals:
    results = web_search_exa(
        query=f"{vertical} {role} founder CEO",
        category="company",
        numResults=20
    )
    # 对每条结果评分

# 第三步：使用 X API 搜索活跃声音
x_search = search_recent_tweets(
    query="prediction markets OR AI tooling OR developer tools",
    max_results=100
)
# 提取并对唯一作者评分
```

## 第二阶段：互动排名

对每位高评分目标，分析用户的社交图谱以寻找最温暖的触达路径。

### 排名模型

1. 提取用户的 X 关注列表和 LinkedIn 人脉
2. 对每位高信号目标，检查是否有共同连接
3. 应用 `social-graph-ranker` 模型对桥接价值评分
4. 按以下因素对共同人脉排名：

| 因素 | 权重 |
|------|------|
| 与目标的连接数量 | 40% — 权重最高，连接越多排名越高 |
| 共同人脉的当前职位/公司 | 20% — 决策者 vs 个人贡献者 |
| 共同人脉的位置 | 15% — 同城 = 更易介绍 |
| 行业对齐度 | 15% — 同垂直领域 = 自然介绍 |
| 共同人脉的 X 账号/LinkedIn | 10% — 可识别性便于外联 |

规范规则：

```text
当用户需要图谱数学本身、作为独立报告的桥接排名，
或明确的衰减模型调优时，使用 social-graph-ranker。
```

在本技能中使用相同的加权桥接模型：

```text
B(m) = Σ_{t ∈ T} w(t) · λ^(d(m,t) - 1)
R(m) = B_ext(m) · (1 + β · engagement(m))
```

解读：
- 第一层：高 `R(m)` 且有直接桥接路径 → 温暖介绍请求
- 第二层：中等 `R(m)` 且有一跳桥接路径 → 条件介绍请求
- 第三层：无可行桥接 → 使用相同潜在客户记录直接冷外联

### 输出格式

```

则先单独运行 `social-graph-ranker`，再将结果反馈到本流水线。
```

## 第三阶段：温暖路径发现

为每位目标寻找最短介绍链：

```
您 ──[关注]──> 共同人脉 A ──[投资于]──> 目标公司
您 ──[关注]──> 共同人脉 B ──[联合创立]──> 目标人物
您 ──[在某活动上认识]──> 活动 ──[同样参与]──> 目标人物
```

### 路径类型（按温暖程度排序）
1. **直接共同人脉** — 您和目标都关注/认识同一个人
2. **投资组合连接** — 共同人脉投资了或为目标公司提供顾问服务
3. **同事/校友** — 共同人脉在同一公司工作过或就读于同一学校
4. **活动重叠** — 双方都参加过同一会议/项目
5. **内容互动** — 目标与共同人脉的内容有互动，或反之

## 第四阶段：富化

对每位合格潜在客户，获取以下信息：

- 全名、当前职位、公司
- 公司规模、融资阶段、近期新闻
- 近期 X 帖子（最近 30 天）——主题、语气、兴趣
- 与用户的共同兴趣（共同关注、相似内容）
- 近期公司动态（产品发布、融资轮次、招聘）

### 富化来源
- Exa：公司数据、新闻、博客文章
- X API：近期推文、简介、粉丝
- GitHub：开源贡献（用于以开发者为中心的潜在客户）
- LinkedIn（通过 browser-use）：完整简介、工作经历、教育背景

## 第五阶段：外联草稿

为每位潜在客户生成个性化外联内容。草稿应与基于来源的声音档案和目标渠道相匹配。

### 渠道规则

#### 电子邮件

- 用于最高价值的冷外联、温暖介绍、投资者外联和合作请求
- 本地桌面控制可用时，默认在 Apple Mail / Mail.app 中起草
- 先创建草稿，除非用户明确要求，否则不自动发送
- 主题行应简洁具体，不求巧妙

#### LinkedIn

- 在目标活跃于此、LinkedIn 上互动图谱关系更强，或邮件可信度低时使用
- 如有 API 访问权限则优先使用
- 否则使用浏览器控制查看简介、近期活动，并起草消息
- 比邮件更简短，避免虚假的职业热情

#### X

- 用于高上下文的运营者、建设者或投资者外联，此类场景中公开发帖行为很重要
- 优先使用 API 访问搜索、时间线和互动分析
- 必要时回退到浏览器控制
- 私信和公开回复应比邮件精炼得多，并引用目标时间线上真实的内容

#### 渠道选择启发式规则

按以下顺序选择一个主要渠道：

1. 通过电子邮件进行温暖介绍
2. 直接邮件
3. LinkedIn 私信
4. X 私信或回复

仅当有充分理由且节奏不会显得骚扰时才使用多渠道。

### 温暖介绍请求（发给共同人脉）

目标：

- 一个明确的请求
- 一个介绍有意义的具体理由
- 如需要，附上易于转发的简介

避免：

- 过度解释您的公司
- 堆砌社会证明
- 听起来像融资模板

### 直接冷外联（发给目标）

目标：

- 以具体近期的内容开场
- 解释匹配的真实原因
- 提出一个低摩擦的请求

避免：

- 泛泛的赞美
- 功能堆砌
- 宽泛的请求如"很乐意联系"
- 强迫性的修辞疑问句

### 执行模式

对每位目标，提供：

1. 推荐渠道
2. 选择该渠道的理由
3. 消息草稿
4. 可选的跟进草稿
5. 如果电子邮件是所选渠道且 Apple Mail 可用，创建草稿而非仅返回文本

如果浏览器控制可用：

- LinkedIn：查看目标简介、近期活动和共同上下文，然后起草或准备消息
- X：查看近期帖子或回复，然后起草私信或公开回复语言

如果桌面自动化可用：

- Apple Mail：创建包含主题、正文和收件人的邮件草稿

未经用户明确批准，不自动发送消息。

### 反模式

- 无个性化的通用模板
- 解释整个公司的长段落
- 一条消息中有多个请求
- 没有具体内容的虚假熟悉感
- 带有明显合并字段的批量发送消息
- 电子邮件、LinkedIn 和 X 复用相同内容
- 平台格式的千篇一律而非作者真实声音

## Agent

本技能在 `agents/` 子目录中包含专用 Agent：

- **signal-scorer** — 按相关性信号搜索并排名目标客户
- **mutual-mapper** — 映射社交图谱连接并寻找温暖路径
- **enrichment-agent** — 获取详细的个人资料和公司数据
- **outreach-drafter** — 生成个性化消息

## 使用示例

```
用户：找出预测市场中我应该联系的前 20 人

Agent 工作流：
1. signal-scorer 使用 Exa 和 X 搜索预测市场领导者
2. mutual-mapper 在用户的 X 图谱中检查共同连接
3. enrichment-agent 获取公司数据和近期活动
4. outreach-drafter 为排名靠前的潜在客户生成个性化消息

输出：带温暖路径的排名列表、声音档案摘要，以及渠道专属外联草稿或应用内草稿
```
