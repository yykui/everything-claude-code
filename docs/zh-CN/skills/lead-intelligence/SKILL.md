# 线索情报

由 Agent 驱动的线索情报管道，通过社交图谱分析和热路径发现，找到高价值联系人、为其评分并与其建立联系。

## 何时激活

- 用户希望在特定行业查找线索或潜在客户
- 为合作、销售或融资构建外联名单
- 研究联系谁以及最佳触达路径
- 用户说"查找线索"、"外联名单"、"我应该联系谁"、"热情引荐"
- 需要按相关性对联系人列表进行评分或排名
- 希望绘制共同联系人地图以找到热情介绍路径

## 工具要求

### 必需
- **Exa MCP** — 对人物、公司和信号进行深度网页搜索（`web_search_exa`）
- **X API** — 粉丝/关注图谱、互动分析、近期动态（`X_BEARER_TOKEN`，以及写入上下文凭证如 `X_CONSUMER_KEY`、`X_CONSUMER_SECRET`、`X_ACCESS_TOKEN`、`X_ACCESS_TOKEN_SECRET`）

### 可选（增强结果）
- **LinkedIn** — 如有直接 API 则使用，否则通过浏览器控制进行搜索、个人资料检查和草稿撰写
- **Apollo/Clay API** — 如用户有权限，可用于富化交叉参考
- **GitHub MCP** — 用于以开发者为中心的线索筛选
- **Apple Mail / Mail.app** — 起草冷邮件或热情邮件而不自动发送
- **浏览器控制** — 当 API 覆盖范围不足或受限时，用于 LinkedIn 和 X

## 外联前的语音建模

不要用通用的销售文案起草对外消息。

只要用户的语音风格很重要，就先运行 `brand-voice`。复用其 `VOICE PROFILE`，而不是在本技能内临时推导风格。

如果有实时 X 访问权限，起草前先拉取近期原创推文。否则，使用提供的示例或可用的最佳仓库/站点素材。

## 第一阶段：信号评分

在目标垂直行业中搜索高信号人物。根据以下维度对每个人进行加权评分：

| 信号 | 权重 | 来源 |
|------|------|------|
| 职位/头衔匹配度 | 30% | Exa、LinkedIn |
| 行业匹配度 | 25% | Exa 公司搜索 |
| 近期话题活跃度 | 20% | X API 搜索、Exa |
| 粉丝数 / 影响力 | 10% | X API |
| 地理位置接近度 | 10% | Exa、LinkedIn |
| 与您内容的互动 | 5% | X API 互动 |

### 信号搜索方法

```python
# 第一步：定义目标参数
target_verticals = ["prediction markets", "AI tooling", "developer tools"]
target_roles = ["founder", "CEO", "CTO", "VP Engineering", "investor", "partner"]
target_locations = ["San Francisco", "New York", "London", "remote"]

# 第二步：使用 Exa 对人物进行深度搜索
for vertical in target_verticals:
    results = web_search_exa(
        query=f"{vertical} {role} founder CEO",
        category="company",
        numResults=20
    )
    # 对每个结果评分

# 第三步：使用 X API 搜索活跃声音
x_search = search_recent_tweets(
    query="prediction markets OR AI tooling OR developer tools",
    max_results=100
)
# 提取并对唯一作者评分
```

## 第二阶段：互动排名

对每个评分目标，分析用户的社交图谱以找到最热情的路径。

### 排名模型

1. 拉取用户的 X 关注列表和 LinkedIn 人脉
2. 对每个高信号目标，检查是否有共同联系人
3. 应用 `social-graph-ranker` 模型对桥接价值评分
4. 按以下因素对互动联系人排名：

| 因素 | 权重 |
|------|------|
| 与目标的联系数量 | 40% — 权重最高，联系最多 = 排名最高 |
| 互动联系人的当前职位/公司 | 20% — 决策者 vs 个人贡献者 |
| 互动联系人的位置 | 15% — 同城 = 更容易介绍 |
| 行业契合度 | 15% — 同一垂直行业 = 自然介绍 |
| 互动联系人的 X 账号 / LinkedIn | 10% — 可识别性，便于外联 |

规范规则：

```text
当用户需要独立的图谱计算、作为单独报告的桥接排名，
或明确的衰减模型调整时，使用 social-graph-ranker。
```

在本技能内部，使用相同的加权桥接模型：

```text
B(m) = Σ_{t ∈ T} w(t) · λ^(d(m,t) - 1)
R(m) = B_ext(m) · (1 + β · engagement(m))
```

解读：
- 第一层：高 `R(m)` 且有直接桥接路径 -> 热情引荐请求
- 第二层：中等 `R(m)` 且有一跳桥接路径 -> 条件引荐请求
- 第三层：无可用桥接 -> 使用相同线索记录直接进行冷外联

### 输出格式

```

如果用户明确希望将排名引擎单独拆出、可视化数学过程，或在完整线索工作流之外对网络进行评分，请先单独运行 `social-graph-ranker`，然后将结果反馈到本管道。
互动排名报告
=====================

    ...
```

## 第三阶段：热路径发现

对每个目标，找到最短的介绍链条：

```
您 ──[关注]──> 互动联系人 A ──[投资了]──> 目标公司
您 ──[关注]──> 互动联系人 B ──[联合创办了]──> 目标人物
您 ──[在……相遇]──> 活动 ──[同样参加]──> 目标人物
```

### 路径类型（按热度排序）
1. **直接互联** — 您和对方都认识同一个人
2. **投资组合联系** — 互动联系人投资了或顾问于目标公司
3. **同事/校友** — 互动联系人在同一家公司工作或就读于同一学校
4. **活动重叠** — 都参加过同一个会议/项目
5. **内容互动** — 目标与互动联系人的内容有互动，或反之

## 第四阶段：富化

对每个合格线索，提取：

- 全名、当前头衔、公司
- 公司规模、融资阶段、近期新闻
- 近期 X 发帖（最近 30 天）——话题、语气、兴趣
- 与用户的共同兴趣（共同关注、相似内容）
- 近期公司事件（产品发布、融资轮次、招聘）

### 富化来源
- Exa：公司数据、新闻、博客文章
- X API：近期推文、简介、粉丝
- GitHub：开源贡献（针对以开发者为中心的线索）
- LinkedIn（通过 browser-use）：完整个人资料、工作经历、教育背景

## 第五阶段：外联草稿

为每个线索生成个性化外联内容。草稿应匹配基于来源的语音档案和目标渠道。

### 渠道规则

#### 邮件

- 适用于最高价值的冷外联、热情引荐、投资人外联和合作请求
- 有本地桌面控制时默认在 Apple Mail / Mail.app 中起草
- 先创建草稿，除非用户明确要求否则不自动发送
- 主题行应简洁明了，避免耍聪明

#### LinkedIn

- 当目标在此平台活跃、LinkedIn 上的互动图谱上下文更强，或邮件置信度较低时使用
- 如有 API 访问权限则优先使用
- 否则使用浏览器控制检查个人资料、近期动态并起草消息
- 比邮件更简短，避免虚假的职业化热情

#### X

- 用于高上下文的运营商、构建者或投资人外联，且目标的公开发帖行为很重要时
- 优先使用 API 进行搜索、时间线和互动分析
- 在需要时回退到浏览器控制
- 私信和公开回复应比邮件紧凑得多，并应引用目标时间线上的真实内容

#### 渠道选择启发式规则

按以下顺序选择一个主要渠道：

1. 通过邮件进行热情引荐
2. 直接发送邮件
3. LinkedIn 私信
4. X 私信或回复

仅在有充分理由且节奏不会显得骚扰时采用多渠道策略。

### 热情引荐请求（给互动联系人）

目标：

- 一个明确的请求
- 一个这次介绍有意义的具体理由
- 如需要，附上易于转发的简介

避免：

- 过度解释您的公司
- 堆砌社会证明
- 听起来像融资模板

### 直接冷外联（给目标人物）

目标：

- 从具体且近期的内容开始
- 解释为什么契合度是真实的
- 提出一个低门槛的请求

避免：

- 泛泛的赞美
- 功能堆砌
- "很想联系"这样宽泛的请求
- 强行设置修辞性问题

### 执行模式

对每个目标，提供：

1. 推荐渠道
2. 该渠道最佳的原因
3. 消息草稿
4. 可选的跟进草稿
5. 如果选择邮件渠道且 Apple Mail 可用，创建草稿而不只是返回文本

如果浏览器控制可用：

- LinkedIn：检查目标个人资料、近期动态和互动上下文，然后起草或准备消息
- X：检查近期推文或回复，然后起草私信或公开回复语言

如果桌面自动化可用：

- Apple Mail：创建带有主题、正文和收件人的邮件草稿

未经用户明确批准不得自动发送消息。

### 反模式

- 无个性化的通用模板
- 解释整个公司的长段落
- 一条消息中包含多个请求
- 没有具体内容的虚假熟悉感
- 带有可见合并字段的批量发送消息
- 在邮件、LinkedIn 和 X 上复用相同文案
- 平台形状的糟糕内容，而非作者真实的语音风格

## 子 Agent

本技能在 `agents/` 子目录中包含专用子 Agent：

- **signal-scorer** — 按相关性信号搜索并排名潜在客户
- **mutual-mapper** — 绘制社交图谱联系并找到热路径
- **enrichment-agent** — 提取详细的个人资料和公司数据
- **outreach-drafter** — 生成个性化消息

## 使用示例

```
用户：帮我找出我应该联系的预测市场领域前 20 人

Agent 工作流：
1. signal-scorer 使用 Exa 和 X 搜索预测市场领袖
2. mutual-mapper 检查用户的 X 图谱以寻找共同联系
3. enrichment-agent 提取公司数据和近期动态
4. outreach-drafter 为排名靠前的线索生成个性化消息

输出：带有热路径的排名列表、语音档案摘要，以及渠道专属的外联草稿或应用内草稿
```
