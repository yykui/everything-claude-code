# 医疗 EMR 开发模式

用于构建电子病历（EMR）和电子健康档案（EHR）系统的开发模式。以患者安全、临床准确性和医生效率为优先。

## 使用时机

- 构建患者就诊工作流（主诉、检查、诊断、处方）
- 实现临床病历记录（结构化 + 自由文本 + 语音转文字）
- 设计带药物相互作用检查的处方/用药模块
- 集成临床决策支持系统（CDSS）
- 构建带参考范围高亮显示的化验结果展示
- 实现临床数据的审计追踪
- 为临床数据录入设计医疗级无障碍 UI

## 工作原理

### 患者安全优先

每个设计决策都必须评估："这会伤害患者吗？"

- 药物相互作用**必须**触发告警，不能静默通过
- 异常化验值**必须**进行视觉标记
- 危急生命体征**必须**触发升级工作流
- 没有审计追踪，不允许修改任何临床数据

### 单页就诊流程

临床就诊应在单页垂直滚动中完成——无需切换标签页：

```
患者信息头部（固定显示——始终可见）
├── 基本信息、过敏史、当前用药
│
就诊流程（垂直滚动）
├── 1. 主诉（结构化模板 + 自由文本）
├── 2. 现病史
├── 3. 体格检查（按系统）
├── 4. 生命体征（自动触发临床评分）
├── 5. 诊断（ICD-10/SNOMED 搜索）
├── 6. 用药（药物数据库 + 相互作用检查）
├── 7. 检查（化验/影像医嘱）
├── 8. 治疗计划与随访
└── 9. 签署 / 锁定 / 打印
```

### 智能模板系统

```typescript
interface ClinicalTemplate {
  id: string;
  name: string;             // e.g., "Chest Pain"
  chips: string[];          // clickable symptom chips
  requiredFields: string[]; // mandatory data points
  redFlags: string[];       // triggers non-dismissable alert
  icdSuggestions: string[]; // pre-mapped diagnosis codes
}
```

任何模板中的红旗症状必须触发可见的、不可关闭的告警——而非 toast 通知。

### 用药安全模式

```
用户选择药物
  → 检查当前用药的相互作用
  → 检查本次就诊用药的相互作用
  → 检查患者过敏史
  → 根据体重/年龄/肾功能验证剂量
  → 如为 CRITICAL 相互作用：完全阻断处方
  → 临床医生必须记录覆盖理由方可绕过阻断
  → 如为 MAJOR 相互作用：显示警告，要求确认
  → 在审计追踪中记录所有告警和覆盖理由
```

危急相互作用**默认阻断处方**。临床医生必须在审计追踪中明确记录覆盖理由。系统绝不静默允许危急相互作用通过。

### 锁定就诊模式

临床就诊签署后：
- 不允许编辑——只能添加附录（独立的关联记录）
- 原始记录和附录均显示在患者时间线中
- 审计追踪记录签署人、签署时间和所有附录记录

### 临床数据 UI 模式

**生命体征展示：** 当前值及正常范围高亮（绿/黄/红），与上次比较的趋势箭头，自动计算的临床评分（NEWS2、qSOFA），内联升级指导。

**化验结果展示：** 正常范围高亮，与上次结果对比，危急值不可关闭告警，采集/分析时间戳，待处理医嘱及预计完成时间。

**处方 PDF：** 一键生成，包含患者基本信息、过敏史、诊断、药物详情（通用名 + 商品名、剂量、给药途径、频次、疗程）、临床医生签名栏。

### 医疗无障碍要求

医疗 UI 的要求比普通 Web 应用更严格：
- 最低 4.5:1 对比度（WCAG AA）——临床医生在不同光照环境中工作
- 大触控目标（最小 44x44px）——适应戴手套或紧急操作
- 键盘导航——适合快速数据录入的高效用户
- 不能仅用颜色作为指示——始终将颜色与文字/图标结合（色盲临床医生）
- 所有表单字段添加屏幕阅读器标签
- 临床告警不得自动关闭 toast——临床医生必须主动确认

### 反模式

- 将临床数据存储在浏览器 localStorage 中
- 药物相互作用检查静默失败
- 危急临床告警使用可关闭的 toast
- 基于标签页的就诊 UI 破坏临床工作流连贯性
- 允许编辑已签署/锁定的就诊记录
- 展示临床数据时不附审计追踪
- 对临床数据结构使用 `any` 类型

## 示例

### 示例 1：患者就诊流程

```
医生为 4521 号患者开启就诊
  → 固定头部显示："Rajesh M，58 岁男性，过敏：青霉素，当前用药：二甲双胍 500mg"
  → 主诉：选择"胸痛"模板
    → 点击症状芯片："胸骨后"、"向左臂放射"、"压榨感"
    → 红旗症状"压榨性胸骨后疼痛"触发不可关闭的告警
  → 体格检查：心血管系统——"S1 S2 正常，无杂音"
  → 生命体征：HR 110，BP 90/60，SpO2 94%
    → NEWS2 自动计算：评分 8，风险高，显示升级告警
  → 诊断：搜索"ACS"→ 选择 ICD-10 I21.9
  → 用药：选择阿司匹林 300mg
    → CDSS 检查与二甲双胍的相互作用：无相互作用
  → 签署就诊记录 → 自此锁定，仅可添加附录
```

### 示例 2：用药安全工作流

```
医生为 4521 号患者开立华法林处方
  → CDSS 检测到：华法林 + 阿司匹林 = CRITICAL 相互作用
  → UI：红色不可关闭的模态框阻断处方
  → 医生点击"记录理由并覆盖"
  → 输入："获益大于风险——已建立 INR 监测方案"
  → 覆盖理由 + 告警存入审计追踪
  → 处方继续进行，附有记录的覆盖理由
```

### 示例 3：锁定就诊 + 附录

```
就诊记录 #E-2024-0891 由 Shah 医生于 14:30 签署
  → 所有字段锁定——不显示编辑按钮
  → "添加附录"按钮可用
  → Shah 医生点击附录，添加："已收到化验结果——肌钙蛋白升高"
  → 新记录 E-2024-0891-A1 关联到原始记录
  → 时间线同时显示：原始就诊记录 + 附录（均带时间戳）
```
