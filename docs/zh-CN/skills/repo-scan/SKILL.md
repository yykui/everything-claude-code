# repo-scan

> 每个生态系统都有自己的依赖管理器，但没有工具能跨越 C++、Android、iOS 和 Web，告诉你：有多少代码是自己写的，哪些是第三方的，哪些是死重。

## 何时使用

- 接手大型遗留代码库，需要结构性概览时
- 重大重构之前 — 识别核心代码、重复代码和冗余代码
- 审计直接嵌入源码（而非通过包管理器声明）的第三方依赖时
- 为 Monorepo 重组准备架构决策记录时

> 安装任何 agent 技能前请先审查源码。

## 核心能力

| 能力 | 描述 |
|------|------|
| **跨栈扫描** | C/C++、Java/Android、iOS（OC/Swift）、Web（TS/JS/Vue）一次性扫描 |
| **文件分类** | 每个文件标记为项目代码、第三方代码或构建产物 |
| **库检测** | 识别 50+ 已知库（FFmpeg、Boost、OpenSSL…）并提取版本 |
| **四级评判** | 核心资产 / 提取合并 / 重建 / 废弃 |
| **HTML 报告** | 带有下钻导航的深色主题交互式页面 |
| **Monorepo 支持** | 层级扫描，包含摘要报告和子项目报告 |

## 分析深度级别

| 级别 | 读取文件数 | 使用场景 |
|------|-----------|---------|
| `fast` | 每模块 1-2 个 | 快速盘点大型目录 |
| `standard` | 每模块 2-5 个 | 包含完整依赖和架构检查的默认审计 |
| `deep` | 每模块 5-10 个 | 额外增加线程安全、内存管理、API 一致性检查 |
| `full` | 所有文件 | 合并前的全面审查 |

## 工作原理

1. **分类代码库界面**：枚举文件，将每个文件标记为项目代码、内嵌第三方代码或构建产物。
2. **检测内嵌库**：检查目录名、头文件、许可证文件和版本标记，以识别打包的依赖项及可能的版本。
3. **为每个模块评分**：按模块或子系统对文件分组，根据所有权、重复程度和维护成本分配四级评判之一。
4. **标记结构性风险**：指出冗余产物、重复包装器、过时的本地化依赖，以及应提取、重建或废弃的模块。
5. **生成报告**：返回简洁摘要及包含每模块下钻功能的交互式 HTML 输出，以便异步审查审计结果。

## 示例

在一个 5 万文件的 C++ Monorepo 上：
- 发现仍在生产环境使用的 FFmpeg 2.x（2015 年版本）
- 发现同一 SDK 包装器被复制了 3 份
- 识别出 636 MB 已提交的 Debug/ipch/obj 构建产物
- 分类结果：3 MB 项目代码 vs 596 MB 第三方代码

## 最佳实践

- 首次审计从 `standard` 深度开始
- 对拥有 100+ 模块的 Monorepo 使用 `fast` 快速盘点
- 对标记需要重构的模块逐步运行 `deep`
- 查看跨模块分析，检测子项目间的重复
