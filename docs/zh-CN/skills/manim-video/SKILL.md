# Manim 视频

在动感、结构和清晰度比真实感更重要的技术解说场景中使用 Manim。

## 何时激活

- 用户需要技术解说动画
- 概念是图表、工作流、架构、指标变化或系统架构图
- 用户需要用于 X 或落地页的简短产品或发布解说视频
- 视觉效果应显得精确，而非泛化的电影风格

## 工具要求

- `manim` CLI 用于场景渲染
- `ffmpeg` 用于后期处理（如需要）
- `video-editing` 用于最终剪辑或精修
- `remotion-video-creation` 用于最终成品需要合成 UI、字幕或额外动态层的情况

## 默认输出

- 简短的 16:9 MP4 文件
- 一张缩略图或海报帧
- 分镜脚本和场景计划

## 工作流

1. 用一句话定义核心视觉主旨。
2. 将概念拆分为 3 到 6 个场景。
3. 决定每个场景要证明什么。
4. 在编写 Manim 代码前先写场景大纲。
5. 先渲染最小可用版本。
6. 渲染效果确认后，再优化排版、间距、色彩和节奏。
7. 仅在有价值时才移交给更广泛的视频工具栈。

## 场景规划原则

- 每个场景只证明一件事
- 避免过于拥挤的图表
- 优先采用逐步展示，而非满屏堆砌
- 用动画解释状态变化，而不仅仅是让屏幕保持动态
- 标题卡应简短且富含信息

## 网络图默认设置

对于社交图谱和网络优化解说：

- 先展示当前图谱，再展示优化后的图谱
- 区分低信号关注噪音与高信号桥接节点
- 高亮显示热路径节点和目标集群
- 如有价值，添加最后一个场景展示形成本技能的自我改进谱系

## 渲染规范

- 除非用户要求竖向，否则默认使用 16:9 横向
- 先进行低质量快速渲染测试
- 在构图和时序稳定后再推进到更高质量
- 导出一张在社交媒体尺寸下清晰可辨的干净缩略帧

## 可复用入门模板

使用 [assets/network_graph_scene.py](assets/network_graph_scene.py) 作为网络图解说的起点。

快速渲染测试示例：

```bash
manim -ql assets/network_graph_scene.py NetworkGraphExplainer
```

## 输出格式

返回：

- 核心视觉主旨
- 分镜脚本
- 场景大纲
- 渲染计划
- 任何后续精修建议

## 相关技能

- `video-editing` 用于最终精修
- `remotion-video-creation` 用于动效密集的后期处理或合成
- `content-engine` 当动画是更大范围发布活动的一部分时
