# Manim 视频

在运动、结构和清晰度比照片真实感更重要的场景中，使用 Manim 制作技术解说动画。

## 何时激活

- 用户希望制作技术解说动画
- 概念是图表、工作流、架构、指标进展或系统架构图
- 用户希望为 X 或落地页制作简短的产品或发布解说
- 视觉效果应感觉精准而非泛泛的电影感

## 工具要求

- `manim` CLI 用于场景渲染
- `ffmpeg` 用于后期处理（如需要）
- `video-editing` 用于最终剪辑或润色
- `remotion-video-creation` 当最终作品需要合成 UI、字幕或额外动效层时

## 默认输出

- 短版 16:9 MP4
- 一张缩略图或海报帧
- 故事板加场景计划

## 工作流

1. 用一句话定义核心视觉论点。
2. 将概念拆分为 3 到 6 个场景。
3. 确定每个场景要证明的内容。
4. 在编写 Manim 代码前先写场景大纲。
5. 先渲染最小可用版本。
6. 渲染成功后，再调整排版、间距、颜色和节奏。
7. 只有在确实增加价值时，才移交给更广泛的视频工具链。

## 场景规划规则

- 每个场景只证明一件事
- 避免过于拥挤的图表
- 优先采用渐进式揭示，而非铺满屏幕的复杂内容
- 用运动来解释状态变化，而非仅为让屏幕保持动感
- 标题卡应简短且富含信息量

## 网络图谱默认设置

用于社交图谱和网络优化解说：

- 先展示当前图谱，再展示优化后的图谱
- 区分低信号关注噪音和高信号桥接节点
- 高亮温暖路径节点和目标聚类
- 如有价值，添加展示启发该技能的自我改进脉络的最终场景

## 渲染规范

- 默认使用 16:9 横版，除非用户要求竖版
- 从低质量冒烟测试渲染开始
- 只有在构图和时序稳定后，才推进到更高质量
- 导出一帧清晰的缩略图，在社交媒体尺寸下可读

## 可复用起始模板

使用 [assets/network_graph_scene.py](assets/network_graph_scene.py) 作为网络图谱解说的起始点。

冒烟测试示例：

```bash
manim -ql assets/network_graph_scene.py NetworkGraphExplainer
```

## 输出格式

返回：

- 核心视觉论点
- 故事板
- 场景大纲
- 渲染计划
- 任何后续润色建议

## 相关技能

- `video-editing` 用于最终润色
- `remotion-video-creation` 用于重度运动后期处理或合成
- `content-engine` 当动画是更大范围发布活动的一部分时
