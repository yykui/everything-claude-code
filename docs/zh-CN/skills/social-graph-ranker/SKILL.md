# 社交图谱排名器

面向网络感知型外展的标准化加权图谱排名层。

以下情况使用本技能：

- 按引介价值对现有共同联系人或连接进行排名
- 映射通往目标列表的温路径
- 衡量一阶和二阶连接之间的桥梁价值
- 决定哪些目标值得温引介，哪些适合直接冷外展
- 独立于 `lead-intelligence` 或 `connections-optimizer` 理解图谱数学

## 何时单独使用本技能

当用户主要需要排名引擎时选择本技能：

- "我的人脉网络中谁最适合帮我引介？"
- "按谁能帮我联系到这些人对我的共同联系人排名"
- "将我的图谱映射到这个 ICP"
- "向我展示桥梁数学"

以下情况请勿单独使用本技能：

- 用户真正需要完整的潜在客户生成和出站序列 -> 使用 `lead-intelligence`
- 用户需要修剪、再平衡和扩展网络 -> 使用 `connections-optimizer`

## 输入

收集或推断：

- 目标人员、公司或 ICP 定义
- 用户在 X、LinkedIn 或两者上的当前图谱
- 加权优先级，例如职位、行业、地理位置和响应率
- 遍历深度和衰减容忍度

## 核心模型

给定：

- `T` = 加权目标集
- `M` = 您当前的共同联系人 / 直接连接
- `d(m, t)` = 从共同联系人 `m` 到目标 `t` 的最短跳数距离
- `w(t)` = 信号评分得出的目标权重

基础桥梁分数：

```text
B(m) = Σ_{t ∈ T} w(t) · λ^(d(m,t) - 1)
```

其中：

- `λ` 是衰减因子，通常为 `0.5`
- 直接路径贡献满分
- 每增加一跳，贡献减半

二阶扩展：

```text
B_ext(m) = B(m) + α · Σ_{m' ∈ N(m) \\ M} Σ_{t ∈ T} w(t) · λ^(d(m',t))
```

其中：

- `N(m) \\ M` 是该共同联系人认识但您不认识的人的集合
- `α` 折扣二阶触达，通常为 `0.3`

响应率调整后的最终排名：

```text
R(m) = B_ext(m) · (1 + β · engagement(m))
```

其中：

- `engagement(m)` 是归一化的响应率或关系强度
- `β` 是互动加成，通常为 `0.2`

解读：

- 第一梯队：高 `R(m)` 且有直接桥梁路径 -> 温引介请求
- 第二梯队：中等 `R(m)` 且有一跳桥梁路径 -> 条件性引介请求
- 第三梯队：低 `R(m)` 或无可用桥梁 -> 直接外展或填补图谱缺口

## 评分信号

在图谱遍历前，根据当前优先级集合对目标进行加权：

- 职位或头衔匹配度
- 公司或行业契合度
- 当前活跃度和时近性
- 地理相关性
- 影响力或触达范围
- 响应可能性

遍历后对共同联系人进行加权：

- 进入目标集的加权路径数量
- 这些路径的直接程度
- 响应率或过往互动历史
- 引介的情境契合度

## 工作流程

1. 构建加权目标集。
2. 从 X、LinkedIn 或两者拉取用户图谱。
3. 计算直接桥梁分数。
4. 为价值最高的共同联系人扩展二阶候选人。
5. 按 `R(m)` 排名。
6. 返回：
   - 最佳温引介请求
   - 条件性桥梁路径
   - 无温路径的图谱缺口

## 相关技能

- `lead-intelligence` 在更广泛的目标发现和外展流水线中使用本排名模型
- `connections-optimizer` 在决定保留、修剪或添加谁时使用相同的桥梁逻辑
- `brand-voice` 应在起草任何引介请求或直接外展前运行
- `x-api` 提供 X 图谱访问和可选的执行路径
