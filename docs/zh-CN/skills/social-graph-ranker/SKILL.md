# 社交图谱排名器

用于网络感知外联的规范加权图谱排名层。

在用户需要以下功能时使用本技能：

- 按介绍价值对现有共同联系人或关系进行排名
- 向目标列表映射温热路径
- 测量一阶和二阶关系间的桥接价值
- 决定哪些目标值得温热介绍，哪些适合直接冷联系
- 独立于 `lead-intelligence` 或 `connections-optimizer` 了解图谱数学原理

## 何时单独使用本技能

当用户主要需要排名引擎时选择本技能：

- "我的人脉中谁最适合帮我做介绍？"
- "按谁能帮我接触到这些人来排名我的共同联系"
- "将我的图谱映射到这个 ICP"
- "给我看桥接数学"

当用户真正需要以下功能时不要单独使用本技能：

- 完整的潜在客户生成和外向销售排序 -> 使用 `lead-intelligence`
- 精简、重平衡和扩展网络 -> 使用 `connections-optimizer`

## 输入

收集或推断：

- 目标人员、公司或 ICP 定义
- 用户在 X、LinkedIn 或两者上的当前图谱
- 加权优先级，如职位、行业、地域和响应度
- 遍历深度和衰减容忍度

## 核心模型

给定：

- `T` = 加权目标集
- `M` = 你当前的共同联系人/直接关系
- `d(m, t)` = 共同联系人 `m` 到目标 `t` 的最短跳数距离
- `w(t)` = 来自信号评分的目标权重

基础桥接分数：

```text
B(m) = Σ_{t ∈ T} w(t) · λ^(d(m,t) - 1)
```

其中：

- `λ` 为衰减因子，通常为 `0.5`
- 直接路径贡献完整价值
- 每增加一跳贡献减半

二阶扩展：

```text
B_ext(m) = B(m) + α · Σ_{m' ∈ N(m) \\ M} Σ_{t ∈ T} w(t) · λ^(d(m',t))
```

其中：

- `N(m) \\ M` 是该共同联系人认识但你不认识的人的集合
- `α` 折扣二阶覆盖，通常为 `0.3`

响应度调整后的最终排名：

```text
R(m) = B_ext(m) · (1 + β · engagement(m))
```

其中：

- `engagement(m)` 是归一化响应度或关系强度
- `β` 是参与度奖励，通常为 `0.2`

解读：

- 一级：高 `R(m)` 且有直接桥接路径 -> 温热介绍请求
- 二级：中等 `R(m)` 且有一跳桥接路径 -> 条件性介绍请求
- 三级：低 `R(m)` 或没有可行桥接 -> 直接外联或填补图谱缺口

## 评分信号

在图谱遍历前，根据当前优先级集合对目标进行加权：

- 职位或头衔匹配度
- 公司或行业契合度
- 当前活跃度和时效性
- 地域相关性
- 影响力或覆盖面
- 回应可能性

遍历后对共同联系人加权：

- 进入目标集的加权路径数量
- 这些路径的直接性
- 响应度或以往互动历史
- 做介绍的情境契合度

## 工作流程

1. 构建加权目标集。
2. 从 X、LinkedIn 或两者拉取用户图谱。
3. 计算直接桥接分数。
4. 为最高价值共同联系人扩展二阶候选人。
5. 按 `R(m)` 排名。
6. 返回：
   - 最佳温热介绍请求
   - 条件性桥接路径
   - 没有温热路径的图谱缺口

## 相关技能

- `lead-intelligence` 在更广泛的目标发现和外联管道中使用本排名模型
- `connections-optimizer` 在决定保留、删除或添加谁时使用相同的桥接逻辑
- `brand-voice` 应在起草任何介绍请求或直接外联之前运行
- `x-api` 提供 X 图谱访问和可选执行路径
