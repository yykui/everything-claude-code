# Connections Optimizer

重新组织用户的人脉网络，而不是将外联视为单向的潜在客户列表。

本技能处理：

- X 关注清理与拓展
- LinkedIn 关注和连接分析
- 审查优先的精简队列
- 添加和关注推荐
- 暖场路径识别
- 以用户真实声音起草 Apple Mail、X DM 和 LinkedIn 消息

## 何时激活

- 用户想要精简 X 关注列表
- 用户想要重新平衡关注对象或联系人
- 用户说"清理我的人脉"、"我应该取消关注谁"、"我应该关注谁"、"我应该重新联系谁"
- 外联质量取决于网络结构，而非单纯的冷门列表生成

## 必要输入

收集或推断：

- 当前优先事项和活跃工作
- 目标职位、行业、地区或生态系统
- 平台选择：X、LinkedIn 或两者
- 禁止触碰名单
- 模式：`light-pass`（轻度处理）、`default`（默认）或 `aggressive`（激进）

若用户未指定模式，使用 `default`。

## 工具要求

### 首选

- `x-api`：用于 X 图谱检查和近期活动
- `lead-intelligence`：用于目标发现和暖场路径排名
- `social-graph-ranker`：当用户希望独立于更广泛的潜在客户工作流对桥接价值评分时使用
- Exa / 深度研究：用于人员和公司信息丰富
- `brand-voice`：在起草外联内容前使用

### 备选

- 浏览器控制：用于 LinkedIn 分析和起草
- 浏览器控制：如果 API 覆盖受限，用于 X 操作
- Apple Mail 或 Mail.app 起草：当邮件是合适渠道时，通过桌面自动化实现

## 安全默认值

- 默认审查优先，绝不盲目自动精简
- X：只精简用户关注的账号，绝不操作粉丝
- LinkedIn：将删除一度连接视为需要手动审查
- 不自动发送 DM、邀请或邮件
- 在任何执行步骤之前先输出排名行动计划和草稿

## 平台规则

### X

- 相互关注比单向关注更稳固
- 未回关的账号可以更积极地精简
- 长期不活跃或已消失的账号应快速浮现
- 互动质量、信号价值和桥接价值比原始粉丝数更重要

### LinkedIn

- 若用户确实有 LinkedIn API 访问权限，优先使用 API
- 缺少 API 访问时，浏览器工作流必须正常工作
- 区分外向关注和已接受的一度连接
- 外向关注可以更自由地精简
- 已接受的一度连接应默认进入审查，而非自动移除

## 模式

### `light-pass`（轻度处理）

- 只精简高置信度低价值的单向关注
- 将其余的放入审查
- 生成小批量添加/关注列表

### `default`（默认）

- 平衡的精简队列
- 平衡的保留列表
- 排名后的添加/关注队列
- 在有用的地方起草暖场介绍或直接外联

### `aggressive`（激进）

- 更大的精简队列
- 对陈旧未回关账号的容忍度更低
- 执行前仍需通过审查门控

## 评分模型

使用以下正向信号：

- 互相关注
- 近期活动
- 与当前优先事项的契合度
- 网络桥接价值
- 职位相关性
- 真实互动历史
- 近期在线情况和响应度

使用以下负向信号：

- 已消失或废弃的账号
- 陈旧的单向关注
- 偏离优先主题群
- 低价值噪音
- 反复未回应
- 有更多优质替代时仍不回关

相互关注和真实暖场路径桥梁应比单向关注受到较少的负面惩罚。

## 工作流程

1. 捕捉优先事项、禁止触碰约束和选定平台。
2. 拉取当前关注/连接清单。
3. 对精简候选人评分，并附明确理由。
4. 对保留候选人评分，并附明确理由。
5. 使用 `lead-intelligence` 加研究资源，对拓展候选人排名。
6. 匹配合适渠道：
   - X DM：用于温暖、快速的社交联络
   - LinkedIn 消息：用于职业图谱的相邻关系
   - Apple Mail 草稿：用于需要更多背景的介绍或外联
7. 起草消息前运行 `brand-voice`。
8. 在任何执行步骤之前，返回审查包。

## 外联规则

- 默认邮件路径是 Apple Mail / Mail.app 草稿创建。
- 不自动发送。
- 根据温度、相关性和背景深度选择渠道。
- 当邮件或不外联才是正确选择时，不强行发送 DM。
- 草稿听起来应像用户本人，而非自动化销售文案。

## 相关技能

- `brand-voice`：可复用的声音档案
- `social-graph-ranker`：独立的桥接评分和暖场路径计算
- `lead-intelligence`：加权目标和暖场路径发现
- `x-api`：X 图谱访问、起草和可选的执行流程
- `content-engine`：当用户同时希望围绕人脉变动发布公开内容时使用
