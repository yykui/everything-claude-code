# 人脉优化器

重组用户的人脉网络，而不是将外联视为单向的潜在客户开发列表。

此技能处理：

- X 关注列表清理和扩展
- LinkedIn 关注和连接分析
- 优先审阅的清理队列
- 添加和关注推荐
- 温情路径识别
- 以用户真实语气生成 Apple Mail、X DM 和 LinkedIn 草稿

## 何时激活

- 用户想要清理 X 关注列表
- 用户想要重新平衡关注或保持连接的对象
- 用户说"整理我的人脉"、"我应该取关谁"、"我应该关注谁"、"我应该重新联系谁"
- 外联质量取决于人脉结构，而非仅仅是冷名单生成

## 所需输入

收集或推断：

- 当前优先事项和活跃工作
- 目标职位、行业、地域或生态系统
- 平台选择：X、LinkedIn 或两者
- 禁止操作名单
- 模式：`light-pass`、`default` 或 `aggressive`

若用户未指定模式，使用 `default`。

## 工具需求

### 首选

- `x-api` 用于 X 图谱检查和近期活动
- `lead-intelligence` 用于目标发现和温情路径排序
- `social-graph-ranker` 当用户希望独立于更广泛的潜在客户工作流对桥接价值进行评分时
- Exa / 深度研究 用于人员和公司信息丰富
- `brand-voice` 在起草外联内容之前使用

### 备选

- 浏览器控制用于 LinkedIn 分析和起草
- 若 API 覆盖受限，浏览器控制用于 X
- 当电子邮件是正确渠道时，通过桌面自动化使用 Apple Mail 或 Mail.app 起草

## 安全默认值

- 默认为优先审阅，绝不盲目自动清理
- X：仅清理用户关注的账号，绝不清理粉丝
- LinkedIn：将删除一度人脉连接视为需要手动审阅的操作
- 不自动发送 DM、邀请或电子邮件
- 在任何执行步骤之前，输出排序后的行动计划和草稿

## 平台规则

### X

- 互相关注比单向关注更稳定
- 未回关的账号可以更积极地清理
- 高度不活跃或已消失的账号应快速浮现
- 互动质量、信号价值和桥接价值比原始粉丝数更重要

### LinkedIn

- 若用户确实拥有 LinkedIn API 访问权限则优先使用 API
- 缺少 API 访问时必须能通过浏览器工作流处理
- 区分单向关注和已接受的一度人脉连接
- 单向关注可以更自由地清理
- 已接受的一度人脉连接应默认为审阅，而非自动删除

## 模式

### `light-pass`（轻量扫描）

- 仅清理高置信度低价值的单向关注
- 将其余的留待审阅
- 生成小型添加/关注列表

### `default`（默认）

- 平衡的清理队列
- 平衡的保留列表
- 排序后的添加/关注队列
- 在有用的地方起草温情介绍或直接外联内容

### `aggressive`（激进）

- 更大的清理队列
- 对陈旧的未回关账号容忍度更低
- 在执行前仍需通过审阅关卡

## 评分模型

使用以下正向信号：

- 互相关注
- 近期活动
- 与当前优先事项的契合度
- 人脉桥接价值
- 职位相关性
- 真实互动历史
- 近期存在感和响应度

使用以下负向信号：

- 已消失或废弃的账号
- 陈旧的单向关注
- 偏离优先事项的话题群
- 低价值噪音
- 多次无响应
- 当存在更多更好替代时仍未回关

互相关注和真实的温情路径桥梁受到的惩罚应少于单向关注。

## 工作流

1. 捕获优先事项、禁止操作约束和所选平台。
2. 拉取当前的关注/连接清单。
3. 为清理候选人评分并附上明确原因。
4. 为保留候选人评分并附上明确原因。
5. 使用 `lead-intelligence` 加上研究工具对扩展候选人进行排序。
6. 匹配正确的渠道：
   - X DM 用于温情、快速的社交接触点
   - LinkedIn 消息用于专业图谱邻近关系
   - Apple Mail 草稿用于需要更多背景的介绍或外联
7. 在起草消息之前运行 `brand-voice`。
8. 在任何执行步骤之前返回审阅包。

## 审阅包格式

```text
CONNECTIONS OPTIMIZER REPORT
============================

Mode:
Platforms:
Priority Set:

Prune Queue
- handle / profile

  confidence:
  action:

Review Queue

  reason:
  risk:

Keep / Protect
- handle / profile
  bridge value:

Add / Follow Targets
- person
  why now:
  warm path:
  preferred channel:

Drafts
- X DM:
- LinkedIn:
- Apple Mail:
```

## 外联规则

- 默认电子邮件路径是 Apple Mail / Mail.app 草稿创建。
- 不自动发送。
- 根据亲密度、相关性和背景深度选择渠道。
- 当电子邮件或不外联是正确选择时，不要强制使用 DM。
- 草稿应听起来像用户本人，而非自动化销售文案。

## 相关技能

- `brand-voice` 用于可复用的语气档案
- `social-graph-ranker` 用于独立的桥接评分和温情路径计算
- `lead-intelligence` 用于加权目标和温情路径发现
- `x-api` 用于 X 图谱访问、起草和可选执行流程
- `content-engine` 当用户还希望围绕人脉变动发布公开内容时
