# 前端设计

当任务不只是"让它能用"，而是"让它看起来有设计感"时使用。

本技能适用于需要清晰设计立场的产品页面、数据面板、应用框架、组件或视觉体系，而非千篇一律的 AI 风格 UI。

## 何时使用

- 从零构建落地页、数据面板或应用界面
- 将平淡的界面升级为有意图、令人印象深刻的设计
- 将产品概念转化为具体的视觉方向
- 实现需要关注字体排版、构图和动效的前端

## 核心原则

选定方向，坚定执行。

安全平均的 UI 通常比风格鲜明、有几个大胆选择的一致美学更糟糕。

## 设计工作流

### 1. 先确定界面框架

在编码之前，先确定：

- 目的
- 受众
- 情感基调
- 视觉方向
- 用户应记住的一件事

可能的方向：

- 极简克制
- 编辑风格
- 工业感
- 奢华感
- 趣味性
- 几何风
- 复古未来主义
- 柔和有机
- 最大化主义

不要随意混合风格。选定一种，干净地执行。

### 2. 构建视觉体系

定义：

- 字体层级
- 颜色变量
- 间距节奏
- 布局逻辑
- 动效规则
- 表面/边框/阴影处理

使用 CSS 变量或项目的 token 系统，使界面在扩展时保持一致性。

### 3. 有意图地排版构图

偏好：

- 不对称，当它能强化层级时
- 叠加，当它能创造深度时
- 强留白，当它能聚焦注意力时
- 仅在产品真正受益于密度时才使用密集布局

除非对称卡片网格明显是正确选择，否则不要将其作为默认方案。

### 4. 让动效有意义

使用动画来：

- 揭示层级
- 分阶段呈现信息
- 强化用户操作
- 创造一到两个令人印象深刻的时刻

不要在各处散布通用的微交互。一个精心设计的加载序列通常比二十个随机悬停效果更有力量。

## 强默认值

### 字体排版

- 选择有个性的字体
- 在适当时将特色展示字体与可读正文字体搭配
- 设计驱动的页面应避免使用通用默认字体

### 颜色

- 坚持清晰的色板
- 一个主色调配合有选择的点缀色，通常比均等权重的彩虹色板更好
- 除非产品确实需要，否则避免老套的紫色渐变白底

### 背景

营造氛围：

- 渐变
- 网格
- 纹理
- 细微噪点
- 图案
- 分层透明度

对于面向产品的页面，纯平背景很少是最佳答案。

### 布局

- 当构图受益时打破网格
- 有意地使用对角线、偏移和分组
- 即使布局非常规，也要保持阅读流程清晰

## 反模式

永远不要默认使用：

- 千篇一律的 SaaS 英雄区块
- 没有层级的通用卡片堆叠
- 没有体系的随机强调色
- 占位符感的字体
- 仅因为动画容易添加而存在的动效

## 执行规则

- 在现有产品中工作时，保留已有的设计体系
- 将技术复杂度与视觉构思相匹配
- 保持无障碍性和响应式设计完整
- 前端在桌面和移动端都应感觉经过深思熟虑

## 质量关卡

交付前：

- 界面有清晰的视觉立场
- 字体排版和间距感觉有意图
- 颜色和动效是对产品的支撑，而非随意的装饰
- 结果不像是通用 AI UI
- 实现是生产级的，而不仅仅是视觉上有趣
