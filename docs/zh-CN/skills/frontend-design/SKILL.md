# 前端设计

当任务不只是"让它跑起来"，而是"让它看起来有设计感"时使用本技能。

本技能适用于需要明确视觉立场而非通用 AI 风格 UI 的产品页面、仪表板、应用外壳、组件或视觉系统。

## 何时使用

- 从零开始构建落地页、仪表板或应用界面
- 将平淡的界面升级为有意图、令人印象深刻的设计
- 将产品概念转化为具体的视觉方向
- 在字体、构图和动效同等重要的前端项目中

## 核心原则

选定一个方向，坚定执行。

平庸安全的 UI 通常比具有少数大胆选择的强烈、统一美学更差。

## 设计工作流

### 1. 先确定界面框架

编码前，明确：

- 目的
- 受众
- 情感基调
- 视觉方向
- 用户应该记住的一件事

可选方向：

- 极简主义
- 编辑风格
- 工业感
- 奢华感
- 趣味性
- 几何感
- 复古未来主义
- 柔和有机
- 最大化主义

不要随意混合方向。选择一个并干净地执行。

### 2. 构建视觉系统

定义：

- 字体层级
- 颜色变量
- 间距节奏
- 布局逻辑
- 动效规则
- 表面/边框/阴影处理

使用 CSS 变量或项目的令牌系统，确保界面随着增长保持统一。

### 3. 有意图地构图

优先选择：

- 不对称布局（当它能增强层级时）
- 元素重叠（当它能创造深度时）
- 强烈的留白（当它能聚焦视线时）
- 密集布局（仅当产品从密度中受益时）

除非明确是最佳选择，否则避免默认使用对称卡片网格。

### 4. 让动效有意义

使用动画来：

- 揭示层级
- 分阶段呈现信息
- 强化用户操作
- 创造一两个令人印象深刻的时刻

不要到处散布通用的微交互。一个精心设计的加载序列通常比二十个随机悬停效果更有力。

## 强烈推荐的默认值

### 字体

- 选择有个性的字体
- 在适当时，用独特的展示字体搭配易读的正文字体
- 当页面以设计为主导时，避免使用通用默认字体

### 颜色

- 坚持一套清晰的调色板
- 一个主色调搭配少量强调色通常比均等权重的彩虹调色板效果更好
- 除非产品真的需要，否则避免陈词滥调的白底紫色渐变

### 背景

营造氛围：

- 渐变
- 网格纹理
- 材质纹理
- 细微噪点
- 图案
- 分层透明度

纯空白背景很少是面向产品页面的最佳答案。

### 布局

- 当构图受益时打破网格
- 有意图地使用对角线、偏移和分组
- 即使布局非常规，也要保持阅读流程清晰

## 反模式

永远不要默认使用：

- 可互换的 SaaS hero 区域
- 无层级的通用卡片堆叠
- 没有系统的随机强调色
- 占位符感觉的字体排版
- 仅因动画容易添加而存在的动效

## 执行规则

- 在现有产品中工作时，保留已建立的设计系统
- 将技术复杂度与视觉创意相匹配
- 保持无障碍性和响应式设计完整
- 前端在桌面和移动端都应感觉经过深思熟虑

## 质量关卡

交付前：

- 界面有清晰的视觉立场
- 字体和间距感觉有意图
- 颜色和动效支撑产品而非随意装饰
- 结果不像通用 AI UI
- 实现达到生产级水准，而非只是视觉上有趣
